## 一、重点事件解读

### 1) GPT-5.4 正式发布，agent 工作流能力被放到台前

- 发生了什么：OpenAI 发布 GPT-5.4（Thinking/Pro 版本），强调原生 computer-use、最高 1M 上下文、Tool Search 与更高 token 效率。
- 为什么重要：这意味着通用模型正从“回答问题”升级为“跨工具执行复杂任务”，企业价值衡量将更偏向端到端完成率。
- 对 alter0 启示：研究与执行任务要默认支持“计划-执行-校验”闭环，并把工具检索/调用成功率纳入日常指标。
- 来源：<https://openai.com/index/introducing-gpt-5-4/>

### 2) Codex Security 进入 research preview，安全 Agent 从概念走向产品化

- 发生了什么：OpenAI 将 Aardvark 升级为 Codex Security，面向企业提供“威胁建模-漏洞验证-修复建议”的端到端链路，并强调降低误报。
- 为什么重要：安全领域正在出现“高置信告警 + 可落地补丁”新范式，减少了传统 SAST/规则扫描噪声。
- 对 alter0 启示：可借鉴“上下文威胁模型 + 自动验证”思路，为高风险任务建立分级门禁和复核模板。
- 来源：<https://openai.com/index/codex-security-now-in-research-preview/>

### 3) Anthropic × Mozilla：两周内发现 22 个 Firefox 漏洞（14 个高危）

- 发生了什么：Anthropic 与 Mozilla 联合披露，Claude Opus 4.6 在两周内发现并推动修复 22 个 Firefox 漏洞（含 14 个高危），多项修复已进 Firefox 148。
- 为什么重要：这标志“AI 辅助安全研究”从实验室走向大规模真实代码库，且在成熟项目上仍能挖出高价值缺陷。
- 对 alter0 启示：对代码与流程任务可引入“最小可复现样例 + 候选修复建议”的输出规范，降低人工验证成本。
- 来源：<https://www.anthropic.com/news/mozilla-firefox-security>、<https://blog.mozilla.org/en/firefox/hardening-firefox-anthropic-red-team/>

### 4) Anthropic 供应链风险争议持续发酵，但云厂商给出“非防务可用”口径

- 对 alter0 启示：日报中涉及政策冲击的事件，需要附带“适用范围/例外条件/执行时间窗”三要素，避免误判。
