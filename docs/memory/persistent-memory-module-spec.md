# Memory 模块持久化说明文档

> 版本：v1.0  
> 状态：Draft / Requirement Baseline  
> 归属需求：R-028

## 1. 目标

建立一份长期稳定、可审阅、可追溯的 Memory 模块说明文档，统一定义记忆体系中文件角色、持久化边界与前端可视化要求。该文档作为后续实现、验收与运维排障的基线。

## 2. 文件职责模型

| 文件 | 角色 | 更新方式 | 生命周期 | 是否默认纳入 Git |
| --- | --- | --- | --- | --- |
| `USER.md` | 用户画像与协作偏好 | 人工维护 | 长期 | 是 |
| `AGENTS.md` | 代理执行规范与边界 | 人工维护 | 长期 | 是 |
| `MEMORY.md` | 长期记忆汇总（跨天沉淀） | 系统/人工维护 | 长期 | 运行态默认否（位于 `.alter0/` 时） |
| `memory/YYYY-MM-DD.md` | 天级记忆日志 | 系统写入 | 天级滚动 | 运行态默认否（位于 `.alter0/` 时） |
| `SOUL.md` | 强制上下文（最高优先级） | 人工维护 | 长期 | 是 |

## 3. 持久化边界

### 3.1 仓库级持久化（可版本追踪）

以下内容作为产品基线，纳入仓库管理：

- 本说明文档：`docs/memory/persistent-memory-module-spec.md`
- 需求文档中的 Memory 需求条目（`R-017`、`R-018`、`R-025`、`R-026`、`R-027`、`R-028`）
- 强制上下文与规范类文件（如 `SOUL.md`、`AGENTS.md`、`USER.md`，取决于部署模式）

### 3.2 运行态持久化（默认不入库）

运行中产生的记忆数据默认写入 `.alter0/memory/*`，遵循本地持久化和重启恢复能力，不作为业务代码提交内容。

- 天级记忆：`.alter0/memory/YYYY-MM-DD.md`
- 长期候选：`.alter0/memory/long-term/YYYY-MM-DD.md`
- 长期主存：`.alter0/memory/long-term/MEMORY.md`

## 4. 前端 Memory 模块可视化要求

在 `Agent -> Memory` 中提供五类只读视图：

1. 长期记忆视图（L1/L2/L3 汇总与关键字段）
2. 天级记忆视图（按日期浏览）
3. 持久化上下文视图（`SOUL.md`）
4. 说明文档视图（本文件）
5. 任务历史视图（任务摘要默认展示，按 `task_id` 下钻日志与产物）

### 4.1 说明文档视图要求

- 支持完整渲染 `docs/memory/persistent-memory-module-spec.md`
- 支持按章节锚点快速定位（目标、文件职责、持久化边界、前端可视化要求）
- 支持空态与加载失败提示
- 保持只读，不提供在线编辑
- 任务历史视图需遵循 `docs/memory/runtime-task-memory-spec.md` 的运行态目录与回链规范

## 5. 与现有需求关系

- `R-017`：会话短期记忆
- `R-018`：跨会话长期记忆
- `R-025`：天级记忆与长期记忆 Markdown 存储
- `R-026`：强制上下文文件
- `R-027`：Agent Memory 模块入口与页面收敛
- `R-028`：补充说明文档持久化与前端可视化契约
- `R-032`：`.alter0` 任务历史存储规范与 Memory 查阅

## 6. 验收口径（文档层）

1. 该文档存在于仓库并可被版本追踪。
2. 需求文档存在 `R-028` 且与本说明文档保持引用一致。
3. 文档明确区分“仓库级持久化”与“运行态持久化”。
4. 文档明确 Memory 页面应展示“说明文档”视图且保持只读。
5. 文档明确任务历史查阅遵循 `.alter0` 运行态规范并支持按 `task_id` 下钻。
