# Persistent Memory Module Specification

> Last update: 2026-03-04
> Status: supported (R-028)

## 1. 模块目标

Persistent Memory Module 负责统一定义并持久化 Agent 运行所依赖的记忆与约束文档，确保跨会话可恢复、可审阅、可追踪，并通过 `Agent -> Memory` 提供只读可视化入口。

## 2. 核心文档职责

| 文档 | 默认位置 | 职责 | 生命周期 | 注入优先级 |
| --- | --- | --- | --- | --- |
| `USER.md` | 仓库根目录（可按租户覆盖） | 维护用户身份、长期偏好、禁用项与交互风格基线 | 用户维度长期维护，按版本迭代 | 中（用户基线） |
| `AGENTS.md` | 仓库根目录或子目录 | 维护代理执行边界、工程约束、提交与协作规则 | 仓库/子目录维度长期维护 | 高（工程执行约束） |
| `MEMORY.md` | `.alter0/memory/long-term/MEMORY.md` | 沉淀跨会话稳定事实、长期偏好与约束摘要 | 持续增量更新，重启后可恢复 | 中（长期记忆） |
| `memory/YYYY-MM-DD.md` | `.alter0/memory/YYYY-MM-DD.md` | 记录当日会话沉淀、压缩摘要与分层条目（L1/L2/L3） | 日维度滚动生成与归档 | 低-中（短中期事实） |
| `SOUL.md` | 仓库根目录 `SOUL.md` | 维护不可违背的强制要求与优先级策略 | 长期维护，支持热更新 | 最高（强制上下文） |

## 3. 文档映射关系

| 维度 | 载体文档 | 模块映射 |
| --- | --- | --- |
| 用户基线 | `USER.md` | 作为用户层语义基线，定义“用户是谁、偏好什么、禁止什么” |
| 代理约束 | `AGENTS.md` | 作为代理执行规则，定义“如何执行、在哪些边界内执行” |
| 长期事实层 | `MEMORY.md` | 对应跨会话长期记忆持久化载体 |
| 天级事实层 | `memory/YYYY-MM-DD.md` | 对应天级记忆持久化载体，按日期切分 |
| 强制策略层 | `SOUL.md` | 对应强制上下文持久化载体，冲突时覆盖普通记忆 |

## 4. Agent Memory 可视化映射

`Agent -> Memory` 页面固定为只读模式，提供以下视图：

1. **长期记忆**：读取 `MEMORY.md`。
2. **天级记忆**：读取 `memory/YYYY-MM-DD.md`（按日期倒序）。
3. **SOUL.md**：读取强制上下文文件。
4. **说明文档**：读取本文件 `docs/memory/persistent-memory-module-spec.md`，用于稳定展示职责与映射定义。

## 5. 一致性约束

1. Persistent Memory 相关载体统一使用 Markdown（`.md`）。
2. `SOUL.md` 与普通记忆链路隔离，不进入自动摘要淘汰流程。
3. `USER.md` / `AGENTS.md` 作为规则与角色定义来源，不由天级记忆压缩任务直接改写。
4. 说明文档视图仅做展示，不提供在线编辑入口。
